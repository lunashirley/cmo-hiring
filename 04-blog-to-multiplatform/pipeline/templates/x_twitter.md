## X (Twitter) Template

### Structural constraints
- **Single tweet:** ≤ 280 characters (hard limit — over this will be truncated by the platform)
- **Thread tweets:** each ≤ 280 characters
- **Thread length:** 3–7 tweets maximum
- **Hashtags:** 0–2 per thread total; never more than 1 per tweet
- **Links:** Count toward character limit. If including a URL, budget ~23 characters for it.

### When to write a single tweet vs. thread
- **Single tweet:** When 1 atom is strong enough to stand alone as a hook/observation
- **Thread:** When 3+ distinct atoms justify sequential delivery; when a stat + context + implication arc exists

### Thread structural pattern
```
1/ [HOOK — specific claim or striking stat. Must work as a standalone tweet.]
2/ [CONTEXT — why this matters, or the "before" state]
3–N/ [BODY TWEETS — one atom per tweet, each delivering a distinct insight]
N-1/ [PROOF or RECAP — bottom line in one tweet]
N/ [CTA — save this, reply with X, link in bio, or follow for more]
```

### Single tweet structural pattern
```
[HOOK stat or insight] + [implication or contrast] + [optional CTA or question]
```

### Minimum atom requirements
- Single tweet: 1 atom minimum
- Thread: at least 3 atoms across the thread
- No thread tweet should repeat an atom already covered

### Tone register
- Terse. Every word must earn its place.
- Active voice, present tense preferred.
- No "excited to share", no corporate-speak.
- Specificity wins: numbers, names, examples > vague generalities.
